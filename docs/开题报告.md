# 基于大语言模型驱动的多智能体生活仿真平台设计与实现

## 开题报告

---

## 一、课题背景与研究意义

### 1.1 研究背景

近年来，大语言模型（Large Language Models, LLMs）在自然语言理解与生成方面取得了突破性进展，为构建具有自主行为能力的智能体（Agent）提供了全新的技术基础。2023 年，Stanford 大学的 Park 等人发表了具有里程碑意义的论文 *Generative Agents: Interactive Simulacra of Human Behavior*，在 UIST 2023 获得 Best Paper，首次展示了 25 个由 LLM 驱动的智能体在虚拟小镇 Smallville 中自主生活、社交、形成记忆与计划的能力。这一工作开辟了「生成式智能体」（Generative Agents）这一全新研究方向，引发了学术界与产业界的广泛关注。

随后，该领域迅速发展：清华大学的 AgentSociety（2025）将智能体规模推至万级，融合心理学与行为科学理论，复现了极化传播、谣言扩散、UBI 政策等社会实验；Altera AI 的 Project Sid（2024）在 Minecraft 环境中实现了 1000+ 智能体的文明模拟，涌现出角色分工、宪法投票、宗教传播等文明行为；Google DeepMind 的 Concordia（2024）提出了通用的生成式 Agent-Based Modeling 框架，引入 Game Master 机制将自然语言动作翻译为环境效果。MIT Media Lab（AAMAS 2025 Oral）进一步提出 LLM Archetypes 方法，将仿真规模推至百万级。

然而，现有研究主要面向学术实验与社会科学研究场景，存在以下局限：（1）API 调用成本高昂（Stanford Smallville 实验花费数千美元），不适合个人用户长期运行；（2）缺乏面向普通创作者的交互界面与用户体验设计；（3）未能覆盖 OC（Original Character，原创角色）创作社区的特定需求，如角色导入、个性化定制、公开画廊等功能。

### 1.2 研究意义

**理论意义**：本课题将心理学 Big Five 人格模型量化为连续参数空间，系统性地建立人格特质到行为决策的端到端映射机制，探索人格驱动的智能体行为建模方法论。同时，提出三层混合 AI 架构（云端大模型 / 本地小模型 / 规则引擎）的分层降级策略，为 LLM 驱动智能体系统的成本优化提供新思路。

**实践意义**：面向 Toyhouse 等 OC 创作社区的数百万用户，提供一个可负担的、可视化的角色生活仿真平台，使创作者能够观察自己设计的角色在虚拟世界中的自主生活、社交互动与关系演化，从而激发创作灵感、深化角色塑造。

## 二、国内外研究现状

### 2.1 生成式智能体（Generative Agents）

生成式智能体是本课题最直接的相关研究方向。Park 等人的开创性工作提出了 Memory Stream + Retrieval + Reflection + Planning 的智能体认知架构，使用 Phaser 框架渲染 2D 虚拟世界，展示了 LLM 驱动的智能体能够自发组织派对、传播信息、形成社交关系等涌现行为。

在此基础上，AgentSociety 融合了心理学、经济学与行为科学理论，对智能体的情感、需求、动机与认知进行了更精细的建模，并在万级规模上复现了多种社会实验。Project Sid 的 PIANO（Parallel Information Aggregation via Neural Orchestration）并行认知架构则证明了大规模智能体文明模拟的可行性。Concordia 提出的 Game Master 机制实现了自然语言动作到环境效果的通用翻译，为构建灵活的仿真框架提供了重要启示。

### 2.2 LLM 智能体记忆机制

记忆系统是智能体实现连贯行为的关键。ACM TOIS（2025）发表的综述系统梳理了 LLM 智能体记忆机制的三阶段演化框架：Storage → Reflection → Experience。MemR³（2024）提出了闭环记忆检索控制机制，路由器在 retrieve / reflect / answer 之间动态选择，提升了记忆利用的效率与准确性。

ScienceDirect（2024）与 Nature HSS Communications（2024）分别从复杂系统与社会仿真的角度，对 LLM + Agent-Based Modeling 的研究范式进行了全面综述。

### 2.3 人格模型与 LLM 行为调控

Big Five（OCEAN）人格模型是心理学领域最广泛接受的人格描述框架，近年来被引入 LLM 智能体研究。arXiv（2025）的研究表明，Big Five 人格特质对 AI 智能体在公共空间的决策行为有显著影响。另一项研究（2025）发现 Agreeableness（宜人性）是促进 LLM 智能体合作行为的主导因子。PMC（2024）的研究则观察到智能体个性能够通过社交互动自发涌现。KDD Workshop（2025）评估了 Big Five 对 LLM 模拟谈判对话的影响。

### 2.4 游戏 AI 与角色行为建模

arXiv（2024）的综述全面梳理了 LLM 在游戏中的应用，涵盖 NPC 行为生成、叙事构建与世界建模。ORIBA（2024）是首个面向 OC（原创角色）的 LLM 角色扮演研究，支持角色深度开发，与本项目的目标用户高度重合。

在决策模型方面，Game AI Pro 中的 Utility Theory 文章阐述了 Utility AI 的理论基础，是 The Sims 系列需求驱动决策系统的理论根基。Hindawi（2011）基于模糊集理论对友谊动态进行了建模，提出了相似性原则驱动的关系演化机制。

### 2.5 大规模仿真与技术架构

MIT Media Lab（AAMAS 2025 Oral）提出的 LLM Archetypes 方法通过聚合相似智能体的行为模式，突破了百万级仿真的规模瓶颈。Sharma & Mehta（2025）提出的 SLM-default, LLM-fallback 分层架构模式——小模型优先、大模型兜底——与本项目的三层 AI 架构理念高度吻合。SimPy 离散事件仿真框架（arXiv, 2024）为构建学术级可复现的仿真引擎提供了成熟的技术基础。

## 三、研究目标与研究内容

### 3.1 研究目标

设计并实现一个基于大语言模型驱动的多智能体生活仿真平台「The World」，使用户能够创建具有 Big Five 人格特质的原创角色，将其放入虚拟世界中自主生活、社交互动、形成记忆与发展关系，并通过实时 2D 可视化界面观察整个过程。核心目标包括：

1. 设计三层混合 AI 架构，在保证关键交互质量的同时将 API 成本降低约 90%
2. 建立 Big Five 人格到行为决策的量化映射机制
3. 实现基于 Utility AI 的自主决策系统与 SimPy 离散事件仿真引擎
4. 构建情感加权的记忆检索系统与多维关系动态演化机制
5. 提供实时 WebSocket 可视化前端与 OC 社区功能

### 3.2 研究内容

#### 3.2.1 三层混合 AI 架构设计

本项目提出云端大模型（Claude API）、本地小模型（Ollama）与规则引擎（Rule-based Templates）的三层混合架构：

- **Tier 1 - Claude API（约 10% 交互）**：用于首次见面、关系里程碑、重大决策等高质量交互场景。采用每日预算控制（默认 20 次/用户/天），使用 claude-sonnet 模型，最大输出 300 tokens。
- **Tier 2 - Ollama 本地模型（约 80% 交互）**：用于日常对话、闲聊等常规交互。通过 httpx 异步客户端调用本地部署的 Llama3 模型，30 秒超时，最大生成 150 tokens，零 API 成本。
- **Tier 3 - 规则引擎（约 10% 交互）**：用于问候、背景 NPC 等简单交互。基于人格特质变体（high/low extraversion、openness 等）选择预定义模板，零延迟、零成本，作为最终兜底。

核心流程为三步管道：`classify_interaction()` 确定理想层级 → `resolve_tier()` 根据可用性进行降级 → `generate_response()` 调用目标层级并捕获异常。

#### 3.2.2 Big Five 人格驱动的行为仿真模型

将心理学 Big Five（OCEAN）模型的五个维度——开放性（Openness）、尽责性（Conscientiousness）、外向性（Extraversion）、宜人性（Agreeableness）、神经质（Neuroticism）——量化为 0-1 连续值，系统性地影响：

- **需求衰减速率**：外向性影响社交需求衰减（`rate × (0.33 + 1.33 × E)`），神经质影响舒适需求衰减（`rate × (0.7 + 0.6 × N)`）
- **活动偏好**：通过 `personality_affinity` 权重实现人格与活动的亲和度匹配
- **社交风格**：影响对话语气、合作倾向、情感表达范围
- **提示词构建**：`personality_to_behavior()` 将人格值转换为自然语言行为画像（speech_style, social_tendency, emotional_range, decision_style, curiosity_level），嵌入 LLM 系统提示
- **规则模板选择**：阈值判定（>0.65 high, <0.35 low）选择对应的人格变体模板

#### 3.2.3 需求-活动-自主决策系统

**6 维需求系统**：hunger、energy、social、fun、hygiene、comfort，各有独立的基础衰减速率与临界阈值，受人格调制。

**21 种活动**：涵盖生存（eat, sleep, nap）、卫生（shower, use_bathroom）、社交（chat, hangout, deep_talk, group_hangout）、娱乐（read, watch_tv, play_games, exercise）、工作技能（work, study, cook）、休闲（relax, meditate, garden）、移动（walk_to）与闲置（idle）。每种活动定义持续时间、需求效果、所需地点类型与人格亲和度。

**Utility-based 决策引擎**：
1. 紧急覆盖：energy < 10 强制睡觉，hunger < 15 强制进食
2. 多维评分：urgency（50%）+ personality affinity（30%）+ 随机因素（20%）+ 社交奖励
3. 跨地点比较：若远程地点最佳得分超过本地 0.15 以上，则选择前往该地点

**SimPy 离散事件仿真**：1 SimPy 单位 = 1 游戏分钟，支持 0.25x-10x 变速。每个角色作为独立的 SimPy 进程运行，选择活动 → 执行（yield timeout）→ 应用效果 → 检查临界值，循环往复。

#### 3.2.4 记忆与关系动态演化机制

**情感加权记忆检索**：记忆存储包含内容文本、类型（interaction, random_event 等）、重要性（0-1）、情感效价（-1 到 1）、上下文（JSONB）与仿真时间戳。检索公式：

```
relevance = importance × exp(-age / 1440) × (1 + |emotional_valence|)
```

其中 1440 为一天的分钟数，实现重要性、时间衰减与情感强度的综合加权。

**多维关系动态**：关系模型包含 friendship_score、romance_score、rivalry_score 三个维度。Big Five 兼容性算法综合宜人性相似度（权重 0.30）、尽责性相似度（0.25）、开放性相似度（0.20）、外向性互补度（0.15）与神经质惩罚因子（双高则 ×0.5）。关系类型随分值演化：stranger → acquaintance（>15） → friend（>40）→ close_friend（>75）→ romantic（romance>50）/ rival（rivalry>50）。里程碑事件在 ±25、±50、±75 分触发。

#### 3.2.5 实时可视化与社区功能

**前端架构**：React 18 + TypeScript 5 + Phaser 3 游戏引擎 + Zustand 状态管理 + Vite 构建工具。

**WebSocket 实时通信**：前后端通过 WebSocket 实时同步时钟状态、角色状态更新、仿真事件、AI 对话等。支持断线自动重连（指数退避 + 抖动，最多 10 次）。

**2D 世界渲染**：2048×1536 像素世界，32px 网格，6 个预定义地点（Home、Central Park、Cozy Cafe、Public Library、Fitness Center、Game Lounge）。角色以圆形头像 + 名字标签显示，支持平滑移动动画与语音气泡。

**社区功能**：公开画廊展示角色、角色创建与管理、跨用户角色互动（规划中）。

### 4.3 数据模型

系统包含 7 张核心数据表：

1. **users**：用户账户（UUID 主键、用户名、邮箱、密码哈希）
2. **characters**：角色信息（人格 JSONB、外观 JSONB、背景 JSONB、仿真状态 JSONB）
3. **worlds**：世界实例（设置 JSONB：tickRate、dayLength、seasonLength、天气开关等）
4. **locations**：地点（类型、坐标、可用活动 JSONB、容量）
5. **memories**：记忆条目（类型、内容、情感效价、重要性、上下文 JSONB、仿真时间戳）
6. **relationships**：关系（友谊/浪漫/敌对三维评分、关系类型、交互摘要 JSONB）
7. **simulation_events**：仿真事件日志（事件类型、数据 JSONB、涉及角色、仿真刻度）

## 五、创新点

1. **三层混合 AI 架构**：首次将云端大模型（Claude）、本地小模型（Ollama）和规则引擎结合为分层降级体系，通过 `classify → resolve → generate` 三步管道，在保证关键交互质量的同时将 API 成本降低约 90%，使个人用户能够长期运行仿真。

2. **Big Five 人格量化驱动的端到端行为映射**：将心理学 Big Five 模型量化为 0-1 连续值空间，系统性地影响需求衰减速率、活动偏好权重、社交风格、LLM 提示词构建和规则模板选择，实现从人格参数到智能体外在行为的完整映射链路。

3. **OC 创作社区定位**：区别于面向研究者的学术仿真平台，本项目面向 OC 创作社区（如 Toyhouse），支持角色创建与导入、公开画廊展示、跨用户角色互动，填补了创作者工具链中角色仿真环节的空白。

4. **情感加权记忆检索**：提出综合重要性、时间衰减与情感强度的记忆检索公式 `relevance = importance × exp(-age/1440) × (1 + |valence|)`，比单纯的 recency 或 relevance 排序更贴近人类记忆的情感偏向特征。

5. **Utility-based 自主决策 + SimPy 离散事件引擎**：融合 The Sims 的 Utility AI 范式（紧急覆盖 + 多维评分 + 跨地点比较）与专业仿真引擎 SimPy，实现学术级可复现的离散事件仿真，每个角色作为独立 SimPy 进程运行。
